# Decoding the Trade-off between Perceptual Realism and Neural Fidelity in fMRI-to-Image Refinement

[![Python](https://img.shields.io/badge/Python-3.11-blue.svg)](https://www.python.org/)
[![PyTorch](https://img.shields.io/badge/PyTorch-2.0+-EE4C2C.svg)](https://pytorch.org/)

This repository contains the official implementation and experimental code for our project **"Decoding the Trade-off between Perceptual Realism and Neural Fidelity in fMRI-to-Image Refinement"**.

## 🧠 Overview

Reconstructing visual stimuli from fMRI signals involves bridging the gap between semantic accuracy and visual realism. While architectures like **MindEye2** excel at semantic fidelity, their base reconstructions often suffer from blurriness, geometric distortions, and missing fine-grained texture details ("diffusion haze"). This project investigates the impact of post-hoc generative refinement, exploring the trade-off between perceptual image quality and the integrity of the original neural signal.

We compare two refinement strategies, both applied to the same MindEye2 base reconstructions:

1. **Baseline (SDXL + GIT captions):** The standard MindEye2 enhancement stage — an SDXL image-to-image loop conditioned on brief GIT-predicted captions at moderate editing strength.
2. **Alternative (SD 1.5 + ControlNet + BLIP-2):** A spatially constrained pipeline using Canny edge maps extracted from the base reconstruction (Gaussian blur 7×7, edge intensity scaled ×0.7) as a structural scaffold, conditioned on dense self-captions generated by BLIP-2 directly from the decoded image (strength=0.75, 30 DDIM steps).

## 📊 Key Findings

Quantitative evaluation on the **Natural Scenes Dataset (NSD)** (Subject 1, 982 test images) reveals a consistent trade-off between perceptual quality and neural fidelity:

| Metric | MindEye2 baseline | SDXL Enhanced | ControlNet (Canny) |
|---|---|---|---|
| PixCorr ↑ | 0.317 | **0.319** | 0.308 |
| SSIM ↑ | 0.414 | **0.439** | 0.424 |
| AlexNet(2) ↑ | **0.957** | **0.957** | 0.953 |
| AlexNet(5) ↑ | **0.989** | 0.986 | 0.982 |
| InceptionV3 ↑ | **0.965** | 0.951 | 0.942 |
| CLIP ↑ | **0.929** | 0.924 | 0.898 |
| EffNet-B ↓ | **0.612** | 0.639 | 0.654 |
| SwAV ↓ | **0.340** | 0.367 | 0.403 |
| Brain Corr. V1 ↑ | **0.320** | 0.303 | 0.278 |
| Brain Corr. higher_vis ↑ | **0.340** | 0.331 | 0.306 |
| Brain Corr. nsd_general ↑ | **0.336** | 0.318 | 0.298 |

**SDXL refinement** improves structural similarity (SSIM +6.2%) at a moderate cost to neural fidelity (−5.2% Brain Corr.). **ControlNet** yields a smaller structural gain (SSIM +2.6%) but a substantially larger drop in neural fidelity (−11.1%). When the base reconstruction contains decoding artifacts, ControlNet's Canny scaffold permanently encodes them into the spatial conditioning signal, and BLIP-2 captions reinforce them as real scene elements — causing the generative backbone to render fMRI noise as photorealistic but semantically incorrect content. Forward/Backward Retrieval scores are identical across all three conditions by construction (they depend on the CLIP-from-brain embedding, not on the refined image).

## 🚀 Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/alessiaianes/mindeye2-controlnet.git
cd mindeye2-controlnet
```

### 2. Environment Setup

Two separate Conda environments are required — one for the MindEye2/SDXL pipeline and one for ControlNet:

```bash
cd MindEyeV2/src

# Environment for MindEye2 base inference and SDXL refinement
conda env create -f env_mindeye.yml

# Environment for the ControlNet refinement pipeline
conda env create -f env_controlnet.yml
```

### 3. Data and Model Downloads

**NSD fMRI data and MindEye2 checkpoints** are downloaded automatically by `recon_inference.ipynb` from the [MindEye2 HuggingFace dataset](https://huggingface.co/datasets/pscotti/mindeyev2). The files downloaded include:

- `datasets/coco_images_224_float16.hdf5`
- `datasets/betas_all_subj01_fp32_renorm.hdf5`
- `datasets/unclip6_epoch0_step110000.ckpt`
- `datasets/gnet_multisubject.pt` (GNet8 encoding model for brain correlation evaluation)
- `train_logs/final_subj01_pretrained_40sess_24bs/last.pth` (pretrained MindEye2 checkpoint)

**ControlNet checkpoint** (`control_sd15_canny.pth`) is downloaded automatically by `canny_recon_inference.py` from HuggingFace at runtime.

**BLIP-2** (`Salesforce/blip2-opt-2.7b`) is downloaded automatically by `canny_recon_inference.py` via the HuggingFace `transformers` library into the directory specified by `--cache_dir` (default: `datasets/`).

## ⚙️ Running the Pipeline

The full pipeline consists of five sequential steps. On a SLURM cluster, all steps can be submitted together via the provided batch script:

```bash
cd MindEyeV2/src
sbatch run_script.slurm
```

To run steps individually (e.g., on a local machine or interactively), follow the order below.

---

**Step 1 — Base inference (MindEye2)**

Run `recon_inference.ipynb` using the `env_mindeye` environment. This produces the base reconstructions and GIT-predicted captions for all test images.

```
Output:
  evals/all_images.pt
  evals/{model_name}/{model_name}_all_recons.pt
  evals/{model_name}/{model_name}_all_predcaptions.pt
  evals/{model_name}/{model_name}_all_clipvoxels.pt
  evals/{model_name}/{model_name}_all_blurryrecons.pt
```

---

**Step 2 — SDXL refinement (baseline)**

Run `enhanced_recon_inference.ipynb` using the `env_mindeye` environment. This applies the standard MindEye2 SDXL image-to-image enhancement conditioned on GIT captions.

```
Input:  evals/all_images.pt
        evals/{model_name}/{model_name}_all_recons.pt
        evals/{model_name}/{model_name}_all_predcaptions.pt

Output: evals/{model_name}/{model_name}_all_enhancedrecons.pt
```

---

**Step 3 — ControlNet refinement (alternative)**

Run `canny_recon_inference.py` using the `env_controlnet` environment. This applies the Canny-guided ControlNet pipeline with BLIP-2 self-captions.

```bash
conda activate env_controlnet
python canny_recon_inference.py \
    --model_name final_subj01_pretrained_40sess_24bs \
    --subj 1 \
    --strength 0.75 \
    --guidance_scale 7.5 \
    --ddim_steps 30 \
    --cache_dir datasets
```

```
Input:  evals/all_images.pt
        evals/{model_name}/{model_name}_all_recons.pt

Output: evals/{model_name}/{model_name}_all_controlnet_canny.pt
        evals/{model_name}/{model_name}_all_blip2captions.pt
        evals/{model_name}/controlnet_canny/comparison_XXXX.png  (per-image visual)
```

---

**Step 4 — Metric evaluation (per method)**

Run `final_evaluations.ipynb` once for each method, passing the corresponding reconstruction file via `--all_recons_path`:

```bash
# Baseline MindEye2
python final_evaluations.py --all_recons_path=evals/{model_name}/{model_name}_all_recons.pt

# SDXL Enhanced
python final_evaluations.py --all_recons_path=evals/{model_name}/{model_name}_all_enhancedrecons.pt

# ControlNet (Canny)
python final_evaluations.py --all_recons_path=evals/{model_name}/{model_name}_all_controlnet_canny.pt
```

```
Output: tables/{model_name}_all_recons.csv
        tables/{model_name}_all_enhancedrecons.csv
        tables/{model_name}_all_controlnet_canny.csv
```

---

**Step 5 — Cross-method comparison**

Run `compare_methods_evaluations.ipynb`. This loads the CSV tables produced in Step 4, computes relative improvements over the baseline, and generates bar chart visualisations.

```
Output: tables/{model_name}_method_comparison.csv
        tables/{model_name}_method_comparison.png
```
