# Decoding the Trade-off between Perceptual Realism and Neural Fidelity in fMRI-to-Image Refinement

[![Python](https://img.shields.io/badge/Python-3.11-blue.svg)](https://www.python.org/)
[![PyTorch](https://img.shields.io/badge/PyTorch-2.0+-EE4C2C.svg)](https://pytorch.org/)

This repository contains the official implementation and experimental code for our project **"Decoding the Trade-off between Perceptual Realism and Neural Fidelity in fMRI-to-Image Refinement"**.

## 🧠 Overview
Reconstructing visual stimuli from fMRI signals involves bridging the gap between semantic accuracy and visual realism. While architectures like **MindEye2** excel at semantic fidelity, their base reconstructions often suffer from "diffusion haze". This project investigates the impact of post-hoc generative refinement, exploring the trade-off between subjective image quality and the integrity of the original neural signal.

We compare:
1. **The Standard MindEye2 Pipeline:** An SDXL-based image-to-image loop conditioned on GIT-predicted captions.
2. **Our Proposed Alternative:** A rigid structural approach using **Stable Diffusion 1.5 + ControlNet** (Canny edge maps) conditioned on dense self-captions generated by **BLIP-2**.

## 📊 Key Findings
Our quantitative and qualitative evaluations on the **Natural Scenes Dataset (NSD)** (Subject 1) reveal that:
* **Unconstrained Refinement (SDXL):** Improves structural similarity (SSIM) and perceptual realism but incurs a moderate decrease in neural fidelity (GNet8 correlation).
* **Rigid Constrained Refinement (ControlNet):** Locks early decoding artifacts into the spatial scaffold and forces the model to render fMRI noise as photorealistic details. This results in a severe drop in neural fidelity, demonstrating that strong spatial conditioning is counterproductive without ground-truth supervision.

## 🚀 Getting Started

### 1. Clone the repository

First, clone the repository to your local machine or cluster:

```bash
git clone https://github.com/alessiaianes/mindeye2-controlnet.git
cd mindeye2-controlnet

```

### 2. Environment Setup

Navigate to the source directory and create the required Conda environments using the provided YAML files:

```bash
cd MindEyeV2/src

# Create the environment for the ControlNet pipeline
conda env create -f env_controlnet.yml 

# Create the environment for the MindEye2 pipeline
conda env create -f env_mindeye.yml 

```

### 3. Data Download

Please refer to the original [MindEye2 repository](https://github.com/MedARC-AI/MindEyeV2) or internal documentation for instructions on downloading the fMRI base reconstructions and the NSD ground-truth dataset required to run the evaluation.

## ⚙️ Running the Project

This project is configured to run on a SLURM-based cluster. To execute the pipelines, navigate to the `src` directory and submit the batch script:

```bash
cd MindEyeV2/src
sbatch run_script.slurm

```

*(Note: The SLURM script is configured to automatically manage the execution of both refinement methods and the metric calculation).*
